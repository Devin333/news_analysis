# News / Tech / Homepage / Trend 排序策略说明

## 概述

系统提供四类主要排序策略，分别针对不同的内容消费场景。每种策略有不同的特征权重配置，以满足特定场景的需求。

## 1. News Ranking（新闻排序）

### 适用场景
- 新闻 Feed
- 板块新闻列表
- 实时资讯流

### 核心理念
新闻的核心价值在于**时效性**和**可信度**。用户期望看到最新、最可靠的信息。

### 权重配置

```
recency:           30%  ← 时效性最重要
source_authority:  15%  ← 来源权威性
trusted_source:    15%  ← 可信来源比例
topic_heat:        10%  ← 话题热度
source_diversity:  10%  ← 多源验证
review_bonus:      10%  ← 审核通过
analyst_importance: 5%  ← 分析价值
trend_signal:       5%  ← 趋势信号
```

### 特点
- 高度重视时效性（30%）
- 来源可信度占比高（30%）
- 支持按板块调整权重

### 变体
- `BoardNewsRankingStrategy`：针对特定板块调整权重
  - AI 板块：提高 analyst_importance 和 trend_signal
  - Research 板块：提高 source_authority
  - Engineering 板块：提高 source_diversity

---

## 2. Tech Ranking（技术排序）

### 适用场景
- 技术 Feed
- 深度技术内容
- 工程实践分享

### 核心理念
技术内容的价值在于**新颖性**和**实用性**。用户期望发现新技术、新方法。

### 权重配置

```
historian_novelty:  20%  ← 新颖性最重要
analyst_importance: 15%  ← 分析价值
trend_signal:       15%  ← 趋势信号
trend:              10%  ← 趋势分数
source_diversity:   10%  ← 多源验证
recency:            10%  ← 时效性（相对较低）
topic_size:         10%  ← 话题规模
review_bonus:        5%  ← 审核通过
source_authority:    5%  ← 来源权威
```

### 特点
- 新颖性权重最高（20%）
- 趋势相关权重高（25%）
- 时效性权重相对较低（10%）

### 变体
- `DeepTechRankingStrategy`：深度技术内容
  - 更高的 historian_novelty（25%）
  - 更高的 source_authority（15%）
- `EngineeringRankingStrategy`：工程实践
  - 平衡 topic_size 和 source_diversity

---

## 3. Homepage Ranking（首页排序）

### 适用场景
- 首页展示
- 精选内容
- 跨板块聚合

### 核心理念
首页需要展示**最重要**、**最有价值**的内容，同时保持**板块平衡**。

### 权重配置

```
homepage_candidate: 25%  ← 首页候选分数
review_bonus:       20%  ← 必须通过审核
analyst_importance: 15%  ← 分析价值
trend_signal:       10%  ← 趋势信号
recency:            10%  ← 时效性
source_diversity:   10%  ← 多源验证
topic_heat:         10%  ← 话题热度
```

### 特点
- 审核是硬性要求（20%）
- 首页候选分数综合多个因素
- 板块平衡机制

### 板块平衡
```python
# 每个板块最多 5 条
max_per_board = 5

# 按分数排序后，依次选取
# 如果某板块已满，跳过该板块的后续内容
```

### 变体
- `DiversifiedHomepageStrategy`：多样性优化
  - 使用 MMR（Maximal Marginal Relevance）算法
  - 平衡相关性和多样性

---

## 4. Trend Ranking（趋势排序）

### 适用场景
- 趋势页面
- 热门话题
- 新兴技术追踪

### 核心理念
趋势内容的价值在于**信号强度**和**发展阶段**。用户期望发现正在崛起的话题。

### 权重配置

```
trend_signal:       30%  ← 趋势信号最重要
trend:              20%  ← 趋势分数
topic_heat:         15%  ← 话题热度
source_diversity:   10%  ← 多源验证
analyst_importance: 10%  ← 分析价值
recency:            10%  ← 时效性
topic_size:          5%  ← 话题规模
```

### 特点
- 趋势相关权重最高（50%）
- 根据趋势阶段额外加分

### 趋势阶段加分
```python
stage_boost = {
    "emerging": 0.3,   # 新兴
    "growing":  0.4,   # 增长（最高加分）
    "peak":     0.1,   # 峰值
    "stable":   0.0,   # 稳定
    "declining": -0.1, # 下降
}
```

### 变体
- `EmergingTrendStrategy`：新兴趋势
  - 更高的 historian_novelty（25%）
  - 更高的 recency（20%）
- `HotTrendStrategy`：热门趋势
  - 更高的 topic_heat（25%）
  - 更高的 topic_size（10%）

## 使用示例

### 获取新闻 Feed
```python
context = RankingContextDTO(
    context_name="news_feed",
    time_window_hours=24,
    max_results=50,
)
ranked = await ranking_service.rank_topics(topic_ids, context)
```

### 获取首页内容
```python
context = RankingContextDTO(
    context_name="homepage",
    time_window_hours=48,
    max_results=20,
    include_unreviewed=False,
    min_item_count=2,
    min_source_count=2,
)
ranked = await ranking_service.rank_topics(topic_ids, context)
```

### 获取趋势
```python
context = RankingContextDTO(
    context_name="trend",
    time_window_hours=72,
    max_results=30,
    include_unreviewed=True,
)
ranked = await ranking_service.rank_topics(topic_ids, context)
```

## 调优建议

1. **新闻场景**：如果用户反馈内容太旧，提高 recency 权重
2. **技术场景**：如果用户反馈内容太浅，提高 analyst_importance 权重
3. **首页场景**：如果某板块内容太少，调整 max_per_board
4. **趋势场景**：如果新兴趋势太少，提高 emerging 阶段加分
