# 成本优化策略说明

## 概述

成本优化模块通过智能策略控制 LLM 和 Embedding 调用成本，在保证质量的前提下降低运营成本。

## 成本埋点

### CostLog 模型

记录每次 API 调用的成本信息：

| 字段 | 说明 |
|------|------|
| component | 组件（llm, embedding, agent） |
| model_name | 模型名称 |
| operation | 操作类型 |
| token_input | 输入 token 数 |
| token_output | 输出 token 数 |
| estimated_cost | 估算成本（USD） |
| latency_ms | 延迟（毫秒） |
| target_type | 目标类型 |
| target_id | 目标 ID |
| agent_name | Agent 名称 |

### 使用方式

```python
from app.monitoring.cost import track_llm_call, CostTracker

tracker = CostTracker()
tracker.start()

with track_llm_call("gpt-4o", "generate", tracker=tracker) as ctx:
    # 调用 LLM
    response = await llm.generate(prompt)
    ctx["input_tokens"] = response.usage.prompt_tokens
    ctx["output_tokens"] = response.usage.completion_tokens

# 获取汇总
summary = tracker.get_summary()
print(f"Total cost: ${summary['total_cost']:.4f}")
```

## 优化策略

### 模型分层 (ModelTier)

| 层级 | 模型 | 适用场景 |
|------|------|----------|
| SMALL | gpt-4o-mini, claude-3-haiku | 简单任务（分类、提取） |
| MEDIUM | gpt-4o, claude-3-sonnet | 中等任务（摘要、分析） |
| LARGE | gpt-4, claude-3-opus | 复杂任务（生成、推理） |

### 任务复杂度映射

```python
TASK_COMPLEXITY = {
```

### 跳过策略

1. **低价值主题跳过**
   - 当 topic_value_score < 0.3 时跳过完整处理
   - 节省大量不必要的 LLM 调用

2. **最近处理跳过**
   - 24 小时内处理过的主题不重复处理
   - 除非主题有实质性变化

3. **Historian 跳过**
   - 主题未变化且最近运行过时跳过
   - 历史上下文通常不需要频繁更新

### 缓存策略

- Writer 输出缓存：主题无变化时复用之前的文案
- 缓存 TTL：48 小时

### 批量优化

- Embedding 批量处理：最大 100 条/批
- 减少 API 调用次数

## Cost-Aware Pipeline

### 配置示例

```python
from app.monitoring.cost_policies import CostPolicyConfig

config = CostPolicyConfig(
    default_tier=ModelTier.MEDIUM,
    use_small_for_simple=True,
    use_large_for_reports=True,
```

```python

pipeline = CostAwarePipeline()
```

## 成本监控

### 按维度聚合

```python
summary = tracker.get_summary()

# 按模型
for model, stats in summary["by_model"].items():
    print(f"{model}: ${stats['cost']:.4f} ({stats['calls']} calls)")

# 按组件
for component, stats in summary["by_component"].items():
    print(f"{component}: ${stats['cost']:.4f}")

# 按 Agent
for agent, stats in summary["by_agent"].items():
    print(f"{agent}: ${stats['cost']:.4f}")
```

### 成本告警

建议设置以下告警：

1. 单次调用成本 > $0.10
2. 单个主题处理成本 > $0.50
3. 日成本 > 预算阈值
4. 异常高的 token 使用量

## 最佳实践

1. **分层使用模型**: 简单任务用小模型，复杂任务用大模型
2. **设置合理阈值**: 根据业务价值调整跳过阈值
3. **监控成本趋势**: 定期检查成本报告
4. **优化 Prompt**: 减少不必要的 token 使用
5. **批量处理**: 尽可能批量调用 Embedding API
