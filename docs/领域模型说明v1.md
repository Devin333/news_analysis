# 领域模型说明 v1

## 概述

本文档描述 NewsAgent 系统的核心领域模型设计。

## 实体关系

```
Source (1) ──────< (N) RawItem
   │                    │
   │                    │ (1)
   │                    ▼
   └──────────< (N) NormalizedItem
                        │
                        │ (N)
                        ▼
                   TopicItem >──────── (N) Topic
```

## 核心实体

### Source（来源）

表示内容来源，如 RSS 订阅、网站、GitHub 仓库等。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| name | str | 来源名称 |
| source_type | enum | 来源类型 (rss/web/github/arxiv) |
| base_url | str | 基础 URL |
| feed_url | str | RSS 订阅地址 |
| priority | int | 优先级 (0-1000) |
| trust_score | float | 可信度 (0-1) |
| fetch_interval_minutes | int | 抓取间隔 |
| is_active | bool | 是否启用 |
| metadata_json | json | 扩展元数据 |

### RawItem（原始内容）

采集器抓取的原始内容，未经处理。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| source_id | int | 来源 ID (FK) |
| external_id | str | 外部 ID |
| url | str | 原始 URL |
| canonical_url | str | 规范化 URL |
| raw_html | text | 原始 HTML |
| raw_json | json | 原始 JSON |
| raw_text | text | 原始文本 |
| fetched_at | datetime | 抓取时间 |
| checksum | str | 内容校验和 |
| parse_status | str | 解析状态 |

### NormalizedItem（标准化内容）

经过解析、清洗、标准化后的内容。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| raw_item_id | int | 原始内容 ID (FK) |
| source_id | int | 来源 ID (FK) |
| title | str | 标题 |
| clean_text | text | 清洗后正文 |
| excerpt | text | 摘要 |
| author | str | 作者 |
| published_at | datetime | 发布时间 |
| language | str | 语言 |
| content_type | enum | 内容类型 |
| board_type_candidate | enum | 板块候选 |
| quality_score | float | 质量分 |
| ai_relevance_score | float | AI 相关性分 |
| canonical_url | str | 规范化 URL |
| metadata_json | json | 扩展元数据 |

### Topic（话题）

聚合相关内容的话题实体。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| board_type | str | 所属板块 |
| topic_type | str | 话题类型 |
| title | str | 话题标题 |
| summary | text | 话题摘要 |
| representative_item_id | int | 代表性内容 ID |
| first_seen_at | datetime | 首次出现时间 |
| last_seen_at | datetime | 最后更新时间 |
| item_count | int | 关联内容数 |
| source_count | int | 来源数 |
| heat_score | float | 热度分 |
| trend_score | float | 趋势分 |
| status | str | 状态 |

### TopicItem（话题-内容关联）

话题与标准化内容的多对多关联。

| 字段 | 类型 | 说明 |
|------|------|------|
| topic_id | int | 话题 ID (PK, FK) |
| item_id | int | 内容 ID (PK, FK) |
| link_reason | str | 关联原因 |
| created_at | datetime | 创建时间 |

## 枚举类型

### SourceType
- `rss` - RSS 订阅
- `web` - 网页
- `github` - GitHub
- `arxiv` - arXiv 论文

### ContentType
- `article` - 文章
- `paper` - 论文
- `repository` - 代码仓库
- `thread` - 讨论帖

### BoardType
- `general` - 通用
- `ai` - AI 相关
- `engineering` - 工程
- `research` - 研究

## Repository 模式

每个实体都有对应的 Repository 实现：

- `SourceRepository` - 来源 CRUD
- `RawItemRepository` - 原始内容 CRUD + 去重查询
- `NormalizedItemRepository` - 标准化内容 CRUD + 候选查询
- `TopicRepository` - 话题 CRUD + 关联管理

## Unit of Work

`UnitOfWork` 类聚合所有 Repository，提供事务边界：

```python
async with UnitOfWork() as uow:
    source = await uow.sources.create(data)
    await uow.commit()
```
