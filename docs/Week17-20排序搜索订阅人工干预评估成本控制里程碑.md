# Week 17-20 排序、搜索、订阅、人工干预、评估、成本控制里程碑

## 概述

Week 17-20 完成了系统从"能产出内容"到"可运营平台"的升级，新增了排序、搜索、订阅、人工干预、评估和成本控制六大核心能力。

## Week 17: Ranking（排序体系）

### 完成内容

1. **Ranking 协议层**
   - `RankingStrategyProtocol`: 排序策略协议
   - `RankingFeatureProviderProtocol`: 特征提供者协议

2. **Ranking DTO**
   - `RankingFeatureDTO`: 排序特征
   - `RankingScoreDTO`: 排序分数
   - `RankedTopicDTO`: 排序后的主题

3. **特征提取层**
   - recency_score: 时效性分数
   - source_diversity_score: 来源多样性
   - trusted_source_score: 可信来源分数
   - topic_size_score: 主题规模
   - trend_signal_score: 趋势信号
   - analyst_importance_score: 分析师重要性
   - historian_novelty_score: 历史新颖性
   - review_pass_bonus: 审核通过加分
   - stale_penalty: 过期惩罚

4. **排序策略**
   - NewsRankingStrategy: 新闻排序（时效性优先）
   - TechRankingStrategy: 技术排序（新意优先）
   - HomepageRankingStrategy: 首页排序（综合平衡）
   - TrendRankingStrategy: 趋势排序
   - ReportSelectionStrategy: 报告选择

5. **Ranking Trace**
   - ranking_log 表记录排序日志
   - 调试 API 支持查看排序原因

## Week 18: Search（搜索体系）

### 完成内容

1. **Search 协议层**
   - `SearchEngineProtocol`: 搜索引擎协议
   - `SearchQueryDTO`: 搜索查询
   - `SearchResultItemDTO`: 搜索结果项
   - `SearchResponseDTO`: 搜索响应

2. **Keyword Search**
   - 标题/摘要搜索
   - 标签过滤
   - 板块/内容类型/日期过滤
   - PostgreSQL 查询构建器

3. **Semantic Search**
   - Query embedding
   - 向量相似度搜索
   - 语义结果合并

4. **Hybrid Search**
   - 关键词 + 语义混合
   - 结果重排序
   - 多种搜索策略（user_search, topic_lookup, entity_lookup）

5. **Search Ranking & Explain**
   - 搜索结果排序
   - 匹配解释字段（matched_by, matched_fields, similarity_score）

## Week 19: Subscription + Human-in-the-loop

### Subscription（订阅系统）

1. **数据结构**
   - subscription 表: 订阅配置
   - subscription_event 表: 匹配事件

2. **订阅类型**
   - query: 关键词订阅
   - tag: 标签订阅
   - entity: 实体订阅
   - topic: 主题订阅
   - board: 板块订阅

3. **匹配引擎**
   - SubscriptionMatcher: 多种匹配方式
   - 匹配策略配置（strict, fuzzy, semantic）

4. **定时任务**
   - SubscriptionMatchJob: 定期扫描新内容
   - 支持 topics, reports, trends 扫描

### Human-in-the-loop（人工干预）

1. **数据结构**
   - editor_action 表: 编辑操作记录

2. **操作类型**
   - approve/reject: 批准/拒绝
   - revise_copy: 修改文案
   - reassign_board: 重新分配板块
   - split_topic/merge_topic: 拆分/合并主题
   - rerun_agent: 重跑 Agent
   - pin/unpin/feature/archive: 内容管理

3. **HITLService**
   - 完整的人工操作服务
   - 操作历史追溯
   - 待处理重跑队列

4. **Admin API**
   - 完整的运营后台接口
   - 支持所有 HITL 操作

## Week 20: Evaluation + Cost Optimization

### Evaluation（评估体系）

1. **指标体系**
   - board_classification_accuracy: 板块分类准确率
   - topic_merge_precision/recall/f1: 主题合并精确率/召回率
   - historian_consistency: 历史学家一致性
   - analyst_value_quality: 分析师质量
   - writer_faithfulness: 写手忠实度
   - reviewer_catch_rate: 审稿人捕获率
   - trend_detection_precision: 趋势检测精确率

2. **Dataset 结构**
   - DatasetLoader: 数据集加载器
   - 标注样本 JSON 格式
   - 支持多种评估类型

3. **Evaluation Runner**
   - 自动化评估执行
   - 阈值检查
   - 结果汇总和建议

### Cost Optimization（成本优化）

1. **成本埋点**
   - cost_log 表: 成本日志
   - CostTracker: 成本追踪器
   - 模型定价配置

2. **成本策略**
   - ModelTier: 模型分层（small/medium/large）
   - 任务复杂度映射
   - 跳过低价值主题
   - 缓存策略

3. **Cost-Aware Pipeline**
   - 根据主题价值选择模型
   - 智能跳过不必要的步骤
   - 批量处理优化
